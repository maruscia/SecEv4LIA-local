# Examples: Ollama Demo

Use `secev examples ollama` to launch a ready-to-run demo in the TUI.
The command preconfigures an Ollama target, validates local prerequisites, and starts an attack flow automatically.

## Prerequisites

1. Install SecEv4LIA:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install git+https://github.com/AISecurityLab/SecEv4LIA.git
```

2. Install and start Ollama:

```bash
ollama serve
```

## Run the Demo

```bash
secev examples ollama
```

The CLI will:

1. Check that Ollama is reachable
2. Verify required models from the demo config
3. Pull missing models automatically
4. Open the TUI and launch the demo

## Demo Configuration

The built-in demo configuration is loaded from the packaged module `secev4lia.examples.ollama.demo`.
In the repository, the reference script is available at `examples/ollama/demo.py`.

You can customize:

- Target model
- Attacker and judge models
- Attack type and goals

## Troubleshooting

- **Ollama not running**: start it with `ollama serve`
- **Missing model**: the CLI pulls missing models automatically
- **TUI startup issue**: run `secev --version` first to confirm installation
